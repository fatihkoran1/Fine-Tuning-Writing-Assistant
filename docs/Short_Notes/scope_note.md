# Scope Note

This project will build a small prototype writing assistant for a low-resource language setting. The goal is to test whether a small model can be fine-tuned with limited data and whether it gives useful suggestions or mainly memorizes training examples.

I will use Guarani Mbya as the real-world inspiration, but I will only use safe public data or simulate the low-resource situation if real language data is not clearly safe.

I will not build a production-ready tool, a perfect language assistant, or a polished UI. The focus is dataset preparation, baseline testing, LoRA fine-tuning, evaluation, memorization checking, and responsible reflection.

The final proof will be a cleaned dataset, baseline outputs, fine-tuning notebook, evaluation table, memorization check, simple Streamlit demo, Git commits, and decision logs.
